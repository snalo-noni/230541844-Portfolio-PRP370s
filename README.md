### Reflection on Coding in Markdown *(STAR Method)*

**Situation**
For my PRP370S module I had to build a digital portfolio on GitHub. I had never used Markdown before so this was something new I had to figure out.

**Task**
I had to write my CV and the rest of my portfolio content using Markdown so that it would display properly on GitHub and GitHub Pages.

**Action**
I looked up how Markdown works and practised using headings, bullet points, tables, and links. I tried different formatting options until things looked the way I wanted. It took some trial and error but I kept going back and fixing things.

**Result**
By the end I had a full portfolio written in Markdown. It is something I will use again because README files on GitHub are always written in Markdown, so it is a useful skill to have as a developer.

### Reflection on Mock Interview Video Experience *(STAR Method)*

**Situation**
As part of work readiness training I had to do a mock job interview and record it. I was nervous because I had not done anything like this before.

**Task**
I needed to answer interview questions on camera about my skills and experience, and then add the video to my portfolio.

**Action**
I prepared by thinking through questions I might be asked about my projects and my background. I practised a few times before recording. On the day I found a quiet spot, dressed neatly, and recorded myself answering the questions as best I could.

**Result**
Watching the video back helped me see where I need to improve, like slowing down when I speak and being more confident. It was uncomfortable at first but it was good practice for real interviews in future.

### Reflection on the Use of GitHub Pages *(STAR Method)*

**Situation**
After finishing my portfolio content I had to make it live on the internet using GitHub Pages. I had only used GitHub for storing code before, so publishing a website from it was new to me.

**Task**
I had to enable GitHub Pages on my repository so that my portfolio would be accessible as a real website with a link I could submit on Blackboard.

**Action**
I went into the settings of my GitHub repository, found the Pages section, and set the source to the main branch. After saving it I waited a few minutes and then refreshed until the live link appeared. I clicked on the link to check that everything showed up correctly.

**Result**
My portfolio went live and I got the link to submit. I learned that GitHub Pages is a free and simple way to host a website, which is something developers use for personal projects and documentation.
